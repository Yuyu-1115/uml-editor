# Use Case F. Resize objects
## Precondition
按鈕 select 被按下的情況。
## Case
1. 使用者將滑鼠移動至某基本物件範圍內。
2. 將此基本物件的 ports 做明確的顯示。
3. 使用者點選該基本物件的任一 port 按住 mouse 左鍵不放。
4. 使用者不放開左鍵，進行拖曳(drag)的動作。
5. 使用者拖曳到另外一個座標 x, y，放開左鍵 (mouse released) 。
6. 該物件重新繪製，系統以滑鼠放開之座標作為該選定 port 的新位置，並根據該位置重新計算出該物件的新大小。
## Alternatives F.1
若使用者點選的是 composite 物件，則無法進行縮放拖曳，放開游標後物件維持原大小與位置不會有任何動作。
## Alternatives F.2
當使用者拖曳 port 發生交叉反向拖曳 （例如：將右下角的 port 拖曳並越過左上角）時，系統應允許此操作。系統需自動重新計算並轉換物件的基準座標（例如：原本拖曳的右下角變成新物件的左上角），並以拖曳起點與終點的絕對差數值來繪製新物件。
## Alternatives F.3
在任何 resize 的過程中，若計算出的新物件寬度或高度小於系統定義的最小尺寸（例如：寬度與高度不得小於 20 pixels），系統應強制將該物件繪製為預設的最小尺寸 （最小尺寸限制應該要能被明顯看出）。
